# Planejamento do Projeto — BOX BEEF Single Page

## 1. Premissas de entrada

O planejamento considera como base obrigatória o Documento de Requisitos — BOX BEEF Single Page já aprovado, mantendo o escopo institucional enxuto, em uma única página, com navegação por âncoras internas, foco em conversão local e abordagem mobile first.

Premissas consolidadas para esta etapa:

* O site será one page, sem páginas secundárias.
* A navegação principal será composta por: Início, Sobre, Cardápio, Happy Hour e Contato.
* O objetivo principal é transformar interesse local em ação rápida: chamar no WhatsApp, consultar o cardápio, ver Instagram e traçar rota.
* O projeto futuro será implementado apenas com HTML, CSS e JavaScript puro.
* Não haverá blog, e-commerce, sistema de reservas, CMS, SPA, bundlers, bibliotecas externas ou integrações complexas.
* A proposta da marca é de boutique de carnes premium acessível com experiência social urbana.
* O conteúdo deve ser escaneável, com CTA principal visível cedo e navegação rápida no celular.
* O cardápio será apresentado de forma institucional e resumida por categorias, sem reproduzir integralmente a experiência de delivery.
* Foram recebidas como insumo desta etapa:

  * logo da BOX BEEF
  * endereço: R. José de Melo, 350 - Parque Francal, Franca - SP, 14403-073
  * telefone/WhatsApp: (16) 99281-0772
  * Instagram: [https://www.instagram.com/boxbeeffranca/](https://www.instagram.com/boxbeeffranca/)
  * horário base informado: 10:00–23:00
  * entrega: 09:00–22:00
  * happy hour: 17:00–23:00
  * prints do cardápio com categorias e itens
* A referência “Box Beef, Chão & Brasa Moema” com happy hour das 17h às 20h não deve ser usada como verdade do projeto, pois indica outra unidade/contexto e conflita com os dados enviados para Franca.

## 2. Arquitetura da single page

A arquitetura do site será linear, orientada à rolagem vertical e dividida em blocos semânticos independentes, cada um com função clara de comunicação e conversão.

Estrutura macro:

* Header fixo ou semi-fixo com logo, menu por âncoras e CTA curto
* Main com cinco seções principais
* Footer institucional simples com reforço de contato e localização

Arquitetura proposta:

1. Header / Navegação
2. Hero / Início
3. Sobre
4. Cardápio
5. Happy Hour
6. Contato e Localização
7. Footer

Função de cada bloco:

* Header: navegação rápida e acesso imediato ao CTA principal
* Hero: apresentar proposta de valor e iniciar conversão
* Sobre: contextualizar marca e posicionamento
* Cardápio: provar variedade, qualidade e acessibilidade percebida
* Happy Hour: destacar experiência social urbana e motivo de visita
* Contato: reduzir atrito para falar, visitar ou seguir a marca
* Footer: fechamento institucional e repetição de links úteis

## 3. Fluxo de navegação por âncoras

A navegação será interna, simples e previsível, usando IDs semânticos nas seções e links âncora no menu principal.

Fluxo principal:

* Início → `#inicio`
* Sobre → `#sobre`
* Cardápio → `#cardapio`
* Happy Hour → `#happy-hour`
* Contato → `#contato`

Fluxos secundários por CTA:

* CTA principal no Hero → link para WhatsApp
* CTA secundário no Hero → âncora para `#cardapio` ou link para Google Maps
* CTA na seção Sobre → âncora para `#cardapio`
* CTA na seção Cardápio → link para WhatsApp
* CTA na seção Happy Hour → link para WhatsApp ou Maps
* CTA na seção Contato → WhatsApp, Instagram e Maps

Regras de navegação:

* O usuário deve alcançar qualquer seção em no máximo um toque a partir do menu.
* Em mobile, o menu deve ser compacto e de acesso imediato.
* A navegação por âncora deve considerar offset visual para não ocultar títulos sob header fixo.
* O item “Contato” deve ser facilmente acessível sem necessidade de rolagem extensa.

## 4. Hierarquia de informação

A hierarquia seguirá lógica de decisão do usuário local, do mais importante para o mais explicativo.

Hierarquia primária:

1. O que é a BOX BEEF
2. Por que visitar
3. O que tem no cardápio
4. Quando ir para happy hour
5. Como entrar em contato e chegar

Hierarquia por dobra:

* Primeira dobra:

  * logo
  * headline
  * proposta de valor
  * CTA principal
  * CTA secundário
* Segunda dobra:

  * explicação curta da marca
  * diferenciais principais
* Terceira dobra:

  * categorias do cardápio
  * destaques institucionais
* Quarta dobra:

  * faixa/área dedicada ao happy hour
  * horários e apelo social
* Quinta dobra:

  * endereço
  * WhatsApp
  * Instagram
  * Maps
  * horário de funcionamento

Ordem de prioridade de elementos em mobile:

1. Headline
2. CTA principal
3. Texto curto de proposta
4. Acesso ao cardápio
5. Acesso ao Maps
6. Diferenciais
7. Happy hour
8. Contato completo

Componentes previstos:

Padronização esperada:

## 9. Decisões técnicas

**Semântica HTML**

* Usar estrutura semântica com `header`, `nav`, `main`, `section`, `footer`.
* Cada seção principal deve possuir `id` para navegação por âncora.
* Títulos devem respeitar hierarquia lógica de `h1`, `h2` e subtítulos.
* Botões que levam a destinos externos podem ser implementados como links estilizados.
* Informações de contato devem usar marcação apropriada para telefone, endereço e links.

**Organização CSS**

* Separar responsabilidades em arquivos simples: reset, variáveis, base, layout, componentes e seções.
* Usar naming consistente e previsível.
* Priorizar CSS enxuto, sem duplicação desnecessária.
* Estruturar estilos mobile first, com media queries progressivas para telas maiores.
* Definir tokens básicos em `variables.css` para cores, espaçamentos, tipografia e raios.

**Organização JS**

* Manter um único arquivo `main.js`.
* Usar JavaScript apenas para interações leves, como:

  * comportamento do menu mobile
  * destaque de navegação ativa, se necessário
  * rolagem suave, se implementada manualmente
* Evitar lógica complexa e dependência de estado elaborado.
* Não usar JavaScript para renderizar conteúdo principal.

**Uso de imagens**

* Usar a logo enviada como base oficial da marca.
* Selecionar poucas imagens fortes e relevantes em vez de muitas imagens pequenas.
* Priorizar imagens reais dos produtos e do ambiente.
* Criar versões otimizadas para web.
* Usar `alt` descritivo e dimensões controladas para evitar layout shift.
* O cardápio institucional não deve depender de galeria extensa nem carrossel obrigatório.

**Links externos**

* WhatsApp deve abrir conversa direta com número configurado.
* Instagram deve apontar para o perfil oficial enviado.
* Google Maps deve apontar para o endereço da unidade de Franca.
* Links externos podem abrir em nova aba por conveniência, com consistência em todo o projeto.

**Responsividade**

* O layout será desenhado primeiro para telas pequenas.
* Conteúdo em coluna única no mobile.
* Expansão para duas colunas apenas onde houver ganho claro em tablet/desktop.
* Botões e links devem ter área de toque confortável.
* Imagens devem escalar sem cortes críticos de informação.

**Navegação por âncoras**

* As âncoras devem funcionar no menu e em CTAs internos.
* O header, se fixo, exige compensação visual no topo das seções.
* A rolagem entre seções deve ser clara e previsível.
* O nome das âncoras deve ser legível e alinhado ao menu:

  * `#inicio`
  * `#sobre`
  * `#cardapio`
  * `#happy-hour`
  * `#contato`
